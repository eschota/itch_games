- Игра: `unsoccer`. В документах она еще обозначена как `v0.0.001`, но

- Локальный server под тестом: временный QA server `127.0.0.1:8788`, запущен

- Production check: `https://io-games.mecharulez.com/unsoccer/`.

## Bugs

- P1 release/version mismatch: `unsoccer/shared/src/index.ts` экспортирует
  `GAME_VERSION = "v0.0.002"`, текущий server из source отвечает `v0.0.002`,
  но package metadata, skills, `client/index.html`, built metadata и видимые
  browser badges все еще показывают `v0.0.001`. В 5-client browser smoke все
  клиенты показывали badge `v0.0.001`, а `/api/health` того же server отвечал
  `v0.0.002`.
- P1 production route not ready: `https://io-games.mecharulez.com/unsoccer/`
  вернул `200`, но это старая generic IO Games страница, а не UnSoccer client.
  `/unsoccer/socket/.wrtc/v2/connections` на простой HEAD вернул `403`; live
  multiplayer routing не доказан.
- P2 требование Producer по графике не выполнено в текущем source: в shared есть
  `DAY_CYCLE_SECONDS = 120`, но текущий client code не импортирует и не
  использует эту константу. Освещение статичное: `HemisphereLight` +
  `DirectionalLight`; проверенного realtime 2-minute day/night cycle, HDR
  environment lighting и reactive sun/lighting pass в tested client source нет.
- P2 clean-checkout server-start risk: `server:unsoccer` билдит только server
  workspace перед `node dist/index.js`. Он зависит от уже существующего
  `@itch-games/unsoccer-shared/dist`. Этот QA-проход прошел после
  `npm run typecheck:unsoccer` / `npm run build:unsoccer`, где shared был
  собран заранее.

## Risks

- In-app browser не держит несколько независимых вкладок. Надежный browser
  multiplayer smoke сделан через временную parent page с пятью iframe-клиентами
  и проверкой authoritative server counters.
- Node geckos client smoke в этой среде недоступен: `@geckos.io/client` под
  Node 24 падает с `ReferenceError: RTCPeerConnection is not defined`, потому
  что нет browser WebRTC runtime.
- Vite build предупреждает, что client JS chunk больше 500 kB после minify:
  `529.96 kB` raw, `136.18 kB` gzip. Build не падает, но это performance /
  delivery watch item.
- Пинки телом и ногами есть в кодовых путях (`body`, `left`, `right`, `head`),
  но этот проход не доказал ball movement для каждого input type визуальными или
  state assertions.

## Ideas

- Добавить `smoke:unsoccer`: собрать shared+server на чистом дереве, поднять
  временный server на свободном порту, открыть 5-iframe browser harness,
  проверить `/api/health` и сохранить screenshot.
- Добавить debug-only client panel или test hook: connected role, server
  version, last action, ball velocity. Тогда QA сможет доказывать kick/body hit
  без доступа к geckos internals.
- Сделать visible version единственным источником из `GAME_VERSION`, включая
  static metadata или generated version stamp, чтобы client HTML, package
  metadata, server health и HUD не расходились.
