# Orbital Courier QA - 2026-07-01 pending v0.0.006

## Scope

- Local pending build: `v0.0.006` from the dirty working tree on `main`.
- Live production route: `https://io-games.mecharulez.com/orbital-courier/`,
  still serving `v0.0.005` at the time of this pass.
- Browser surfaces: in-app browser desktop `1280x720`, mobile `390x844`,
  cache-busted local harness, local catalog page, production game route.
- Package surface: `python3 tools/package_itch.py`.

## Bugs

- No blocking functional gameplay bug found in the tested pending build.
- P3 UI/accessibility: the visible `Start Run` / `Restart Run` button still
  computes `pointer-events: none` through the overlay. Functional restart passed
  via canvas/document fallback, but the visible button is not the real pointer
  target, so the UI semantics remain misleading.

## Risks

- Pending `v0.0.006` is not live yet. Production health/status and public route
  still report `v0.0.005`.
- Web Audio sound was not audibly verified in this automated pass. Static code
  now wires `run_start`, `run_complete`, `shield_hit`, `near_miss`, and
  `core_collect` events, and browser gameplay stayed console-clean, but the
  in-app browser evaluation context cannot reliably inspect page-owned globals.
- The game script URL remains `./src/main.js`. During local iteration the
  in-app browser kept a stale module for the same URL until a cache-busted
  harness was used. Before release, consider asset versioning or cache policy
  validation so players do not keep stale JS across behavior changes.
- Working tree was actively changing during QA due parallel role work. Final
  release validation should be repeated after the code is committed and deployed.

## Ideas

- Add a tiny automated smoke script that checks visible version, console errors,
  package zip root, and leaked internal files.
- Add a QA/debug-visible audio state or event counter in non-production debug
  mode so automated tests can verify sound hooks without relying on audible
  playback.
- Make the overlay button the actual pointer target and keep the document/canvas
  fallback only as iframe resilience.
